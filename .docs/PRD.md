# 프로젝트 요구사항 정의서 (PRD) - 마이링크 (MyLink)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **서비스 목적**: 사용자가 분산된 작업물, 소셜 링크, 포트폴리오를 하나의 페이지로 모아서 쉽게 공유할 수 있는 통합 프로필 링크 서비스
- **대상 사용자**: 
  - 개발자 (이력서, GitHub, 블로그, 포트폴리오 링크 통합)
  - 크리에이터 및 인플루언서 (다양한 소셜 채널 및 작업물 통합 공유)

## 2. 디자인 및 기술 스택
- **사용자 인증**: Firebase Authentication (Google 소셜 로그인 전용)
- **데이터베이스**: Firebase Firestore (서브 컬렉션 구조 활용)
- **UI / 디자인 시스템**: shadcn/ui 컴포넌트 및 Tailwind CSS 활용 (테마 기능 없이 단일 시스템으로 유지)

## 3. 핵심 기능 목록

### 필수 기능 (Must-Have)
> 초기 MVP(Minimum Viable Product) 출시에 반드시 포함되어야 할 핵심 기능입니다.
1. **회원 인증 및 계정 관리**
   - Firebase 기반 Google 소셜 로그인 (간편하고 안전한 인증)
   - **디스플레이 네임(`displayName`)**을 활용한 고유 URL 슬러그 설정 (`domain.com/displayName` 형식 사용)
2. **프로필 관리**
   - **디스플레이 네임(`displayName`) 및 실제 이름(`username`)**: URL 및 메인 표시명으로 쓰이는 디스플레이 네임과 더불어 사용자의 실제 이름(유저네임) 데이터도 함께 관리 및 수정할 수 있습니다.
   - 자기소개글(Bio) 입력 및 수정 기능
   - *(이미지 직접 자동/수동 업로드 기능은 제공하지 않음)*
3. **핵심 링크 관리 (Dashboard)**
   - 링크 생성: 제목(Title)과 URL 주소 추가
   - **인라인 편집(Inline Editing)** 및 삭제: 별도의 페이지 이동이나 팝업 없이 리스트 항목의 텍스트를 클릭하여 바로 수정
   - **자동 파비콘 가져오기**: 구글 파비콘 API(`https://www.google.com/s2/favicons?domain=...`)를 사용하여 대상 서버의 아이콘을 자동으로 렌더링
4. **SNS 공유 최적화**
   - 프로필 링크를 공유할 때 잘 보이도록 Open Graph(썸네일, 제목, 설명) 기본 지원

### 보류 및 향후 구현 예정 (추가 기획)
1. **링크 클릭수 통계**: 개별 링크의 클릭 조회수(Click Counts)를 파악하는 기능
2. **특수 링크 임베드 위젯**: 유튜브, 깃허브 등 확장 정보 포함 기능

### ⛔ 제외 스펙 (명시적 배제 항목)
- 프로필 이미지 업로드
- 링크 순서 변경 (드래그 앤 드롭)
- 링크 활성화/비활성화 (토글) 버튼
- 커스텀 테마 선택 기능
- 관리자 화면의 모바일 뷰 미리보기 (Preview 패널) 기능

## 4. 데이터베이스 및 동작 시나리오 초안

### 4.1 계정, URL 정책 및 데이터 모델링
- 구글 소셜 로그인 완료 시 사용자 지메일 주소의 앞부분(예: `abc@gmail.com`의 `abc`)을 추출하여 **디스플레이 네임(`displayName`)**의 초기값으로 자동 설정합니다.
- 이 `displayName`은 화면에 노출되는 가장 중요한 표시명이자, **URL 슬러그**(`domain.com/displayName`)로 곧바로 사용되므로 중복될 수 없습니다.
- URL 슬러그 역할을 겸하는 `displayName`과 별도로, 사용자의 **실제 이름(`username`)** 정보도 함께 수집하고 관리합니다.
- **Firestore 서브 컬렉션 패턴**: 사용자 문서(`users` 컬렉션의 id) 하위에 `links` 서브 컬렉션(Sub-collection)을 두어 각 링크 데이터를 안전하게 구조화하여 저장합니다.

### 4.2 관리자 대시보드 (Admin Dashboard)
- 링크 목록 상의 각 항목을 클릭하면 즉시 입력 폼 상태(인라인 에디터)로 전환되어 빠르고 직관적으로 수정할 수 있도록 개발합니다.
- 복잡한 모바일 사이드 뷰 없이 깔끔하고 큼직하게 폼 요소에만 집중하는 UX를 부여합니다.
